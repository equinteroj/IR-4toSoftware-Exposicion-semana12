## Equipo A: StarUML

### Expositor: Steven Díaz

#### Teoría
* ¿Qué es Model Driven Development (MDD)?
* Módulos: MDE, MDA, MDD, M2M, M2T.
* Herramienta CASE para el modelado UML.
* Soporta extensiones para la generación de código.

#### Práctica
* Abrir proyecto.
* Cómo generar código.
* Instalar módulo de generación.

### Expositor: Escudero

#### Teoría
* Integración con UML 2.5.
* Generación de código C#.
* Interfaz intuitiva.
* Arquitectura del subsistema de modelado.
* Fases y operaciones.

#### Práctica
* Modificar elementos.
* Ingeniería inversa.
* Transformación de código a diagrama.

### Expositor: Erick Mera

#### Teoría
* Representación mediante diagramas UML.
* Ventajas: multiplataforma, ligero, rápido, open source.
* Interfaz madura, cumplimiento de estándares y exportación.
* Traducción de diagramas UML a código C#.

#### Práctica
* Navegación en el entorno.
* Verificación de código generado.
* Importar clases a Visual Studio.

### Expositor: Mesías Jhon

#### Teoría
* Desafíos: curva de aprendizaje e integración de utilidades.
* Niveles de compatibilidad.
* Comparativa de rendimiento, generación de código y manejo de proyectos.
* Usabilidad: StarUML vs Visual Paradigm.

#### Práctica
* Automatización de código.
* Selección de directorio de salida.

#### Teoría
* Ciclo integral de modelado.
* Trazabilidad en proyectos.
* Repositorios centralizados.

#### Teoría

* Respaldo institucional.

#### Práctica
* Configuración para generación de código.
* Selección de lenguaje.
* Exportación de código fuente.
* Ingeniería inversa para reconstruir el diagrama.
